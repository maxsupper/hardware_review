# 引脚复用关系检查规则

> 适用场景：从 EDIF 网表检查芯片引脚复用（Mux）配置，验证接口信号是否正确映射
>
> **前置依赖**：本文件是 [引脚核对规则.md](./引脚核对规则.md) 的子集。接口→Bank 归属→VCCIO 域→电压的综合性推断由父文件完成。
>
> **⛔ 启动前必须先读取数据**：
> 1. 读取 `.sisyphus/temp/g0_ic.msg` 获取 IC 清单和手册状态
> 2. 读取 `.sisyphus/temp/vccio.msg` 获取全部 VCCIO 域→电压映射。**禁止在本文件内重新到 EDN 追踪电压**。
> 3. 若上述文件不存在，回到对应前级步骤先生成。
>
> 本文件在此基础上做 Mux 维度的专项检查（三种冲突、速查表、固件一致性）。VCCIO 表、portRef 解析基础均继承自父文件。
>
> ⛔ **G0 前置依赖**：本规则中的 Mux 选项对照和功能验证依赖芯片数据手册中的引脚复用表。执行前必须已完成 `hardware-reviewer.md` 中定义的 Step 0（G0 手册搜索），芯片复用表必须以手册原文为准。若手册 MISSING，Mux 判定结论必须用 ⚫UNVERIFIED 标注。

---

## 核心原则

**一条 portRef 行 = 一个物理引脚的全部复用选项。**

EDIF 网表中的 `portRef` 列出了该引脚可以配置的所有功能。只有出现在这个列表中的功能，芯片才能通过配置寄存器选择。不在列表中的功能，该引脚无法实现。

---

## 数据来源

EDIF 网表中，每个 `portRef` 行格式：

```
portRef FUNC_A_M0  FUNC_B_M1  FUNC_C_M2  FUNC_D_M3  GPIOx_y_z_D  (instanceRef U1000)
```

| 组成部分 | 含义 | 示例 |
|----------|------|------|
| `FUNC_A_M0` | 功能 A，Mux 选项 0（默认） | `UART3_TX_M0` |
| `FUNC_A_M1` | 功能 A，Mux 选项 1 | `UART3_TX_M1` |
| `_M2` / `_M3` | Mux 选项 2/3 | `PWM0_CH2_M2` |
| 无后缀 | 专用功能，不经 Mux | `SDMMC0_CLK` |
| `GPIOx_y_z_D` | GPIO 标识 + IO 域 | `GPIO2_A4_D_J` |

---

## GPIO 标识解读

```
GPIO2_A4_D_J
  │    │  │ └── IO 域后缀（决定电压）
  │    │  └──── 驱动类型（D=推挽驱动, U=上拉）
  │    └─────── 引脚编号（A0~D7）
  └──────────── Bank 编号（0~6）
```

## 检查流程

### 第一步：提取端口复用表

从 EDN 中提取每个引脚的完整 `portRef` 行，按 Mux 分类：

```
解析逻辑：
  for line in edn:
      if "portRef" in line and "U1000" in line:
          funcs = line 按空格拆分
          for f in funcs:
              if f.endswith("_M0"):  → Mux0 功能
              if f.endswith("_M1"):  → Mux1 功能
              if f.endswith("_M2"):  → Mux2 功能
              if f.endswith("_M3"):  → Mux3 功能
              if f.startswith("GPIO"): → GPIO 标识
              else: → 专用功能（非 Mux 功能）
```

### 第二步：逐接口信号核对

对每个接口信号：

```
1. 追踪物理连接 → 得到 SoC 引脚名
2. 查该引脚的 Mux 列表 → 接口需要的功能在列表中吗？
3. 确认 Mux 选项编号 → 固件/BSP 需要配置对应的 Mux
4. 检查 VCCIO 域电压 → 与外设电平匹配吗？
```

### 第三步：检查三种冲突

#### 冲突类型一：专用功能与 GPIO 互斥

```
GPIO2_A5_D_J:
  Mux0:  UART3_CTSN
  Mux3:  UART4_TX
  专用:  SDMMC0_CMD  ← 如果使用了 SD 卡，该引脚不能做 GPIO

判断：在 BSP/设备树中检查 SDMMC0 是否使能
```

#### 冲突类型二：同引脚多功能

```
GPIO5_D3_D_L:
  Mux1:  UART3_CTSN
  Mux2:  SPI1_MISO
  专用:  VO_LCDC_CLK, PWM1_CH0_M1

一个引脚只能选一种功能。如果 UART3_CTSN 使能，则 SPI1_MISO 不可用。
```

#### 冲突类型三：同功能跨域 Mux

```
UART3_TX 可映射到三个不同引脚、不同电压：

  M0: GPIO2_A3_D_J → VCCIO2 = 3.3V  ← 本设计使用
  M1: GPIO5_D4_U_L → VCCIO5 = 1.8V
  M2: GPIO6_C2_D_M → VCCIO6 = 1.8V

如果外设需要 1.8V 但选了 M0，则电压不匹配。
如果外设需要 3.3V 但选了 M1/M2，同样不匹配。
```

## 最终交付检查清单

- [ ] 每个接口信号的物理连接已追踪到 SoC 引脚
- [ ] 所需功能已确认存在于该引脚的 Mux 列表中
- [ ] Mux 选项编号已记录（供 BSP/设备树配置）
- [ ] 专用功能与 GPIO 冲突已排查
- [ ] 跨域 Mux 电压已匹配外设电平
- [ ] VCCIO 域电压已通过网表供电网络确认
- [ ] 网名标签与实际 GPIO Bank 编号一致
- [ ] 固件/BSP 中的 Mux 配置与硬件一致
